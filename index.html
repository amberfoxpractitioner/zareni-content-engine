Global Rules

Voice and Tone

Write as though sitting across from the reader at a kitchen table over a cup of tea
Always use contractions when natural. Address reader as you throughout
Never lecture. Never perform. Always talk to one specific person
Warm without being soft. Direct without being clinical. Confident without being preachy
Every sentence must earn its place. Never pad with filler
Never use AI-sounding phrases, hollow affirmations, or generic wellness language

Hard Prohibitions

Never use dashes of any kind including em dashes, hyphens as separators, or regular dashes
Never bold body text. Headings and subheadings only
Never use the word healing. Use supporting the body, finding real answers, or getting to the root
Never use the word coping. Use managing symptoms or navigating this
Never use you are not broken or any variation of that construction
Never use New Age, spiritual, metaphysical, or woo woo language
Never make promises, bold claims, or outcome guarantees of any kind
Never include H1 in post body. WordPress generates H1 automatically
Never refer to scanning modality by device brand name. Always use bio resonance scanning
Never use you have got this or I see this all the time in my practice
Never recommend allopathic practitioners as primary path

Structure Requirements

Open every post with a hook. Interweave emotional thread throughout. Close the hook at the end
Weave Amber's origin story naturally. Never repeat same detail or phrasing twice in one post
All H2 and H3 headings framed as questions or promises wherever possible
One pillar per topic. One problem, one angle, one explanation per post
Keyword research and explicit approval required before any net new post
Take heart may appear maximum once per post. Never force it
Closing paragraph always warm, human, complete. Never sells anything
Kit opt-in [convertkit form=9245579] placed once mid post after action steps, never at end
Minimum 2000 words. Yoast Readability green light standards throughout
Always end with medical disclaimer in italics

CTA Hierarchy

Primary: toolkit first
Secondary: bio resonance scan session ($397)
Non-negotiable floor: lead magnet opt-in always

Google Drive Setup Instructions
Step 1. Go to console.cloud.google.com and sign in with your Google account.
Step 2. Click Create Project. Name it Zareni Content Engine. Click Create.
Step 3. In the left menu click APIs and Services, then Library. Search for Google Drive API and click Enable.
Step 4. Click APIs and Services, then Credentials. Click Create Credentials, then API Key. Copy the key shown.
Step 5. Paste your API key into the Knowledge Base tab under Google Drive API Key and click Save.
Step 6. For each document you want in your knowledge base, open it in Google Drive and copy the document ID from the URL. The ID is the long string of letters and numbers between /d/ and /edit.
Step 7. Add each document in the Knowledge Base tab. Core documents pull on every run. Pillar documents are selected per run in the Generator.
Step 8. Click Test Drive Connection in the Knowledge Base tab to confirm everything is working.
